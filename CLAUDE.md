# funding-news-bot プロジェクト指示書

このファイルは、資金調達ニュース自動投稿Botの「動作ルール」です。
Claudeはこのプロジェクトで作業するとき、必ず最初にこのファイルを読みます。

## このBotの目的

平日の夕方18時に、日本のスタートアップの資金調達ニュースを集めて、
Substack記事の下書きとX(旧Twitter)用の下書きを自動で作り、Slackに届ける。
（18時に実行する理由: スタートアップは調達ニュースを16〜17時に出すことが多いため。）

運用者: Apricot Ventures / chayano@apricot.vc

## 運用方式（半自動）

Botは「下書きを作ってSlackに届ける」ところまでを担当する。
実際のSubstack投稿・X投稿は、運用者が下書きを確認したうえで手動で行う。
理由: 資金調達ニュースは数字・社名の正確さが命なので、投稿前に人が必ず確認する。

## 平日18時の処理フロー

1. **リサーチ** — PR TIMESで資金調達ニュースを検索する。
   - 検索する言葉は `config.json` の `search.keywords` を使う。
   - 詳しい手順は `prompts/research.md` に従う。
2. **絞り込み** — 集めたニュースから記事対象を選ぶ。
   - 対象: エクイティ調達 と デット調達（`config.json` の `filter.funding_types`）。
   - 日本のスタートアップに限定。海外企業は除外。
   - `config.json` の `filter.exclude` に当てはまるものは除外。
3. **裏取り** — 調達額・調達ラウンド・投資家・ソースURL を確認する。
   - 数字や固有名詞は必ず元のプレスリリースで確認する。推測で書かない。
4. **Substack記事の下書き作成** — 手順は `prompts/article.md` に従う。
   - `reference/format_notes.md` の型に沿って記事下書きのテキストを作る。
   - 出力先は `output/today/`。Botはこの段階では投稿しない。
5. **X下書き生成** — 手順は `prompts/x_draft.md` に従う。
   - メイン投稿＋企業ごとのリプライをテキストで作る。
   - 出力先は `output/today/`。
6. **Slackに通知** — 作った下書き（Substack・X）をSlackの指定チャンネルに投稿する。
   - 投稿先は `config.json` の `notify.slack_channel`。
   - 運用者はSlackで下書きを確認し、Substack投稿・X投稿を手動で行う。

## 記事のトーン・フォーマット

- 記事のトーンや書き方は、`reference/` フォルダに入れた過去記事を読んで学ぶ。
- 過去記事がある場合は、その構成・文体・長さに合わせる。
- 過去記事がまだ無い場合は、事実中心の簡潔なニュース速報スタイルで書く。

## 守るルール

- 事実の正確さを最優先する。調達額・社名・投資家名は元ソースで確認する。
- 確認できない情報は書かない。憶測を断定で書かない。
- 各ニュースには必ずソースURL（元のプレスリリース）を添える。
- BotはSubstack・Xへの投稿を行わない。下書きを作りSlackに届けるところまで。
- 実際の投稿は、運用者が下書きを確認してから手動で行う。

## ファイルの役割

- `config.json` — 検索キーワード・投稿先などの設定。
- `prompts/research.md` — リサーチの指示文。
- `prompts/article.md` — Substack記事の指示文。
- `prompts/x_draft.md` — X下書きの指示文。
- `reference/` — トーンの参考にする過去記事を入れる。
- `output/today/` — 当日生成した記事・下書き。
- `output/archive/` — 過去に生成した分の保管。
