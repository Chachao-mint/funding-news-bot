# 記事作成プロンプト（article.md）

このファイルは、処理フローの「ステップ4 記事作成・投稿」で従う指示文です。
記事のフォーマットは `reference/format_notes.md` に従う。

## 記事作成手順（Substack上で操作）
1. `config.json` の `output.substack_home_url` を開き、Substackのマイページにアクセスする。
2. ホームから、過去に投稿した資金調達ニュース記事を1つ選び、Duplicate（複製）する。
3. 複製した記事の中身を、当日のリサーチメモ（`output/today/`）の内容に差し替える。
4. 差し替えが完了したら内容を確認し、そのページから投稿する。
5. 投稿した記事の控えを `output/today/` にテキストで保存する。

## 差し替えの具体ルール（reference/format_notes.md 準拠）
- **タイトル**: 「M/D(曜日) N件の資金調達ニュース」を当日の日付・件数に差し替える。
- **タイトル直後の番号リスト**: 当日の対象企業名に差し替える（`1/ 社名` 形式）。
- **各社セクション**（過去記事のセクションを上書きする）:
  1. 番号＋企業名
  2. アイキャッチ画像 → PR TIMESプレスリリース冒頭の画像に差し替える
  3. 画像下の文章 → PR TIMESプレスリリースのタイトルを一字一句そのまま引用
  4. ソースURL → 当日のPR TIMES URL
  5. `▼引受先` → 投資家名（エクイティ調達）
  6. `▼融資先` → 銀行名（デット調達の場合のみ）
- 1案件にエクイティとデットが両方ある場合は `▼引受先` と `▼融資先` を両方併記する。
- 当日の件数が過去記事と異なる場合は、セクションを追加・削除して件数を合わせる。
- 末尾の締め（Xアカウント誘導・購読促進）は過去記事のまま残す。

## 守ること
- 数字・社名・投資家名は、裏取り済みのリサーチメモの値だけを使う。
- 過去記事の体裁・レイアウトは変えない。中身の情報だけを入れ替える。
- 投稿前に、調達額・社名・ソースURLに誤りがないか最終確認する。
