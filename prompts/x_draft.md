# X下書きプロンプト（x_draft.md）

このファイルは、処理フローの「ステップ5 X下書き生成」で従う指示文です。
X（旧Twitter）への投稿は手動。ここでは下書きを作るところまで。

## ゴール
当日の資金調達ニュースを、Xにスレッド形式で投稿できる下書きとして作る。
内容はSubstack記事と同じニュースを元にする。

## Xの投稿フォーマット（@Chayano0810 の実例から学習）
Xは「メイン投稿1件 ＋ 企業ごとのリプライ」のスレッド構成。

### メイン投稿
```
本日（M/D)の資金調達ニュース！
N件でした！

1/ 社名
2/ 社名
3/ 社名
4/ 社名

#今日の資金調達ニュース
```
- 1行目: `本日（M/D)の資金調達ニュース！`（M/Dは当日の日付）
- 2行目: `N件でした！`（Nは当日の対象件数）
- 空行をはさんで、企業名の番号リスト（`1/ 社名` 形式・番号順）
- 空行をはさんで、ハッシュタグ `#今日の資金調達ニュース`

### リプライ（企業ごとに1件）
メイン投稿へのリプライとして、企業ごとに「番号」と「PR TIMESのリンク」を投稿する。
- 1社＝1リプライ。番号順に並べる。
- リプライの中身: その企業の番号 と PR TIMESのソースURL。

## ルール
- 絵文字は使わない。
- 数字・社名は、裏取り済みのリサーチメモの値だけを使う。
- メイン投稿の番号・社名は、Substack記事のタイトル直後の番号リストと一致させる。

## アウトプット
メイン投稿と全リプライを `output/today/` にテキストで保存する。
運用者がそれをコピーして、Xに手動で投稿する（メイン投稿 → 各リプライの順）。
